使用方法：
	1、在该文件夹下打开控制台命令窗口（shift+鼠标右键）
	2、在控制台窗口输入：AgvPlanner.exe  ./config/配置文件名（ 例如： ./AgvPlanner.exe  ./config/car_config_03_04.xml）
	3、回车运行

运行结果说明：
                 运行后将生成的带时间窗的轨迹规划信息文件将在出现config 文件夹中， 其中，
              （1）与配置文件同名但后缀为txt，表示以三段式命名站点的时间窗格式文件，方便人分析路径情况，例如：car_config_03_04.txt。
              （2）与配置文件同名自动追加_index.txt 后缀格式的文件，用于可视化显示，例如：car_config_03_04_index.txt 。


config文件夹说明：
               该文件下面放在测试向量（*.xml文件）及运行结果（*.txt文件）。
               配置文件解释：（例如：car_config_03_04.xml）
	1、agv标签下的name标签可以不用管，start_point意为小车起点，	end_point为小车的最终目标点。
	2、type为 小车类型（0为agv，1为叉车）。	
	3、car_v为车速，车速建议设置为符合实际情况（例如 0.4到1）。
	4、middle_point点主要是用于单车多任务，为中间站点（用于小车装货卸货）

file文件夹说明：
                此文件夹下面的文件是通过json地图提取出的信息，供路径规划使用，请勿删除或修改。


注意事项：
	1、在测试多任务时，不要把同时将多个车的end_point设置在同一个断头路里，防止其他车进不去
	2、注意输入的站点名应该是存在的
	3、注意文件名输入要正确。
	4、使用生成的带有index的txt文件跑可视化。
